MySQL语法查漏补缺

练习使用的数据库为随身物品推销商订单录入系统，包括6个表：供应商表（vendors）、产品目录表（products）、顾客表（customers）、订单表（orders）、订单详情表(orderitems)、产品注释表（productnotes）

表1 Vendors（供应商表）

属性名 数据类型 可否为空 含 义
vend_id int(11) 否 唯一的供应商ID
vend_name char(50) 否 供应商名
vend_address char(50) 是 供应商地址
vend_city char(50) 是 供应商的城市
vend_state char(5) 是 供应商的州
vend_zip char(10) 是 供应商的邮政编码
vend_country char(50) 是 供应商的国家

  
表2 Products（产品目录表）

属性名 数据类型 可否为空 含 义
prod_id char(10) 否 唯一的供应商ID
vend_id int(11) 否 供应商名
prod_name char(255) 否 供应商地址
prod_price decimal(8) 否 供应商的城市
prod_desc text(0) 是 供应商的州

  
表3 Customers（顾客表）

属性名 数据类型 可否为空 含 义
cust_id int(11) 否 唯一的顾客ID
cust_name char(50) 否 顾客名
cust_address char(50) 是 顾客的地址
cust_city char(50) 是 顾客的城市
cust_state char(5) 是 顾客的州
cust_zip char(10) 是 顾客的邮政编码
cust_country char(50) 是 顾客的国家
cust_contact char(50) 是 顾客的联系名
cust_email char(255) 是 顾客Email

  
表4 orders（订单表）

属性名 数据类型 可否为空 含 义
order_num int(11) 否 唯一订单号
order_date datetime(0) 否 订单日期
cust_id int(11) 否 订单顾客ID

  
表5 orderitems（订单详情表）

属性名 数据类型 可否为空 含 义
order_num int(11) 否 订单号
order_item int(11) 否 订单物品号
prod_id char(10) 否 产品ID
quantity int(11) 否 物品数量
item_price datetime(8) 否 物品价格

  
表6 productnotes（产品注释表）

属性名 数据类型 可否为空 含 义
note_id int(11) 否 唯一注释ID
prod_id char(10) 否 产品ID
note_date datetime(8)(0) 否 增加注释的日期
note_text text(11) 是 注释文本
